0.9.29 enthält Fehlerkorrekturen

0.9.29 enthält Korrekturen für verschiedene Trac Tickets, inclusive einem Workaround für falsch komprimierte Nachrichten im nativem Datenstrom. Wir unterstützen nun NTP via IPv6. Wir haben vorläufigen Docker Support inkludiert. Wir haben jetzt übersetzte Man-Pages. Wir reichen jetzt den Same-Origin Referer Header durch den HTTP Proxy durch. Weitere Java 9 Anpassungen sind auch enthalten, jedoch empfehlen wir noch nicht den generellen Gebrauch von Java 9.

Wer immer empfehlen wir allen Benutzern, auf diese Version zu aktualisieren. Der Beste Weg, dem Netzwerk zu helfen und sicher zu bleiben, ist die aktuelleste Version zu nutzen.

I2P-Bote 0.4.5 korrigiert eine Sicherheitslücke, die in allen vorherigen Versionen des i2P-Bote-Plugins vorhanden war. Die Android-App war nicht betroffen.

Beim Anzeigen von E-Mails waren die meisten Felder gekapselt. Jedoch waren die Dateinamen der Anhänge nicht gekapselt und konnten so zum Einbringen von Schadcode in Browser mit aktiviertem Javascript genutzt werden. Dieses wurde nun korrigiert und zusätzlich wurden Content Security Policies auf allen Seiten implementiert.

Mit Einspielen der Mitte Februar erwartetem I2P-Aktualisierung 0.9.29 werden alle I2P-Bote Plugins automatisch nach einem I2P Neustart aktualisiert. Dennoch empfehlen wir aus Gründen der Sicherheit eine Manuelle Aktualisierung laut den Instruktionen auf der Installations Seite, falls sie I2P-Bote bis zum nächsten I2P Update nutzen möchten.

0.9.26 enthält Aktualisierungen zur Kryptographie, Verbesserungen in der Debian Paketerstellung und weitere Bug Fixes.

0.9.26 enthält eine große Überarbeitung unserer nativen Kryptobibliothek, ein neues Addressbuch Abo Protokoll mit Signaturen und große Verbesserungen im Debian/Ubuntu Paket System.

Im Bereich Krypto haben wir auf GMP Version 6.0.0 aktualisiert und somit Unterstützung für neuere Prozessoren inkludiert, dieses wird die Kryptooperationen deutlich verschneller. Mit dem Nutzen von GMP Funktionen mit konstanter Zeitnutzung verhindern wir Side-Channel Angriffe. Achtung! Diese GMP Änderungen sind nur in neuen Installationen und in den Debian/Ubuntu Paketen enthalten, in der Version 0.9.27 werden wir diese auch in dem automatischen im-Netz-Update integrieren.

Für die Debian/Ubuntu Pakete haben wir diverse Abhängigkeiten hinzugefügt, unter anderem Jetty8 und GeoIP, damit der äquivalente code aus unserem Paket entfernt werden konnte.

Ebenso sind eine Menge Bugfixes enthalten, unter anderem ein fix für einen Timer-Bug, der Probleme mit der Stabilität und Performanceeinbussen über die Zeit einbrachte. Wie immer empfehlen wir den Usern, diese Version 0.9.26 einzuspielen. Dieses ist der Beste Weg um dem Netzwerk zu helfen und sicher zu bleiben.

Aufräumarbeiten

Mein Signierschlüssel ist seit über 2 Jahren (Version 0.9.9) in den Router Updates, somit sollte das Update gleich funktionieren wie die anderen auch, solange ihr Router aktuelle genug ist. Falls Sie jedoch eine Version vor 0.9.9 nutzen, müssen sie manuell auf die aktuelle Version aktualisieren. Das Update hierzu kann hier, heruntergeladen werden, ein Anleitung finden Sie dort. Nach einem manuellen Update findet Ihr Router wieder die aktuellen Updates und aktualisiert sich automatisch.

Falls Sie I2P mit einem Paket Manager installiert haben, ändert sich für Sie nichts und Sie können wie gewohnt aktualisieren.

Aktualisierungsdetails

Die Migration der RouterInformationen auf die neuen, stärkeren Ed25519 Signaturen funktioniert gut, die Hälfte des Netzwerkes ist schätzungsweise schon migriert. Diese Version beschleunigt den Migrationsvorgang. Um die Belastung im Netzwerk gering zu halten, wird Ihr Router nur zu einer geringen Wahrscheinlichkeit bei einem Neustart migriert. Nach der Migration werden Sie für eine gewisse Zeit eine geringere Bandbreite sehen, da der Router sich neu ins Netzwerk integrieren muss.

Beachten Sie, das dieses die letzte Version sein wird, die Java 6 unterstützen wird. Bitte steigen Sie auf Java 7 oder 8 um, sobald es ihnen möglich ist. Wir arbeiten schon daran, I2P auf Java 9 lauffähig zu haben. Dieses wird in der nächsten Version integriert sein.

Wir haben auch einige kleinere Verbesserungen in I2PSnark eingebaut, ebenso eine neue Seite in der Router Konsole, welche die älteren Neuigkeiten Einträge anzeigt.

Wir empfehlen wie immer, baldmöglichst auf die neue Routerversion zu aktualisieren. Ihrer eigenen Sicherheit und dem Netz ist am besten gedient, wenn Sie stets die neueste stabile Version von I2P verwenden.

0.9.22 mit Fehlerkorrekturen und dem Start der Migration zur Nutzung von Ed25519

0.9.22 enthält Korrekturen zum I2PSnark Problem der kurz vorm Ende festhängenden Torrents und beginnt die Migration der Router zu neueren, sicheren Ed25519 Signaturen. Um die Belastung des Netzwerkes gering zu halten, wird der Router bei jedem Restart nur zu einer geringen Wahrscheinlichkeit zu Ed25519 migriert. Nach erfolgter Umstellung wird die Bandbreite des Routers für einige Tage geringer sein, da der Router sich mit seiner neuen ID neu ins Netzwerk integrieren muss. Falls dieses problemlos funktioniert, beschleunigen wir den Migrationsprozess beim nächsten Release.
