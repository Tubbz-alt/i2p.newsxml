Ştiri şi actualizări de router

I2P-Bote 0.4.4 fixes a security vulnerability present in all earlier versions of the I2P-Bote plugin. The Android app was not affected.

A lack of CSRF protection meant that if a user was running I2P-Bote and then loaded a malicious site in a browser with JavaScript enabled, the adversary could trigger actions in I2P-Bote on behalf of the user, such as sending messages. This might also have enabled extraction of private keys for I2P-Bote addresses, however no proof-of-exploit was tested for this.

All I2P-Bote users will be upgraded automatically the first time they restart their router after I2P 0.9.28 is released in mid-December. However, for safety we recommend that you follow the instructions on the installation page to upgrade manually if you plan on using I2P or I2P-Bote in the intervening time. You should also consider generating new I2P-Bote addresses if you regularly browse sites with JavaScript enabled while running I2P-Bote.

0.9.27 contains bug fixes

0.9.27 contains a number of bug fixes. The updated GMP library for crypto acceleration, which was bundled in the 0.9.26 release for new installs and Debian builds only, is now included in the in-network update for 0.9.27. There are improvements in IPv6 transports, SSU peer testing, and hidden mode.

We updated a number of plugins during I2P Summer and your router will automatically update them after restart.

Ca de obicei, vă recomandăm ca toți utilizatorii să actualizeze la această versiune. Cel mai bun mod de a ajuta rețeua și a rămâne securizat este de a rula ultima versiune.

0.9.26 contains crypto updates, Debian packaging improvements, and bug fixes

0.9.26 contains a major upgrade to our native crypto library, a new addressbook subscription protocol with signatures, and major improvements to the Debian/Ubuntu packaging.

For crypto, we have upgraded to GMP 6.0.0, and added support for newer processors, which will speed up crypto operations considerably. Also, we are now using constant-time GMP functions to prevent side-channel attacks. For caution, the GMP changes are enabled for new installs and Debian/Ubuntu builds only; we will include them for in-net updates in the 0.9.27 release.

For Debian/Ubuntu builds, we have added dependencies on several packages, including Jetty 8 and geoip, and removed the equivalent bundled code.

There's a collection of bug fixes also, including a fix for a timer bug that caused instability and performance degradations over time. As usual, we recommend that all users update to this release. The best way to help the network and stay secure is to run the latest release.

0.9.25 conține SAM 3.3, coduri QR, și erori remediate

0.9.25 conține o nouă versiune majoră a SAM, v3.3, pentru a sprijini aplicațiile Multiprotocol sofisticate. Aceasta adaugă codurile QR pentru partajarea adreselor de servicii ascunse cu alte persoane și imagini "identicon" pentru adrese vizuale distinctive.

Am adăugat o nouă pagină de configurare "familie router", în consolă, pentru a face mai ușor de a declara că grupul dvs. de routere este administrat de o singură persoană. Există mai multe modificări pentru a crește capacitatea rețelei și, sperăm, a îmbunătăți construirea tunelelor cu succes.

Ca de obicei, vă recomandăm ca toți utilizatorii să actualizeze la această versiune. Cel mai bun mod de a ajuta rețeua și a rămâne securizat este de a rula ultima versiune.

0.9.24 conține mai multe remedieri de erori și creștere a vitezei

0.9.24 conține o nouă versiune de SAM (v3.2), precum și numeroase remedieri de erori dar și îmbunătățiri ale eficienței. Rețineți că această versiune este prima care necesită Java 7. Vă rugăm să actualizați la Java 7 sau 8, cât mai curând posibil. Router-ul nu se va actualiza automat dacă utilizați Java 6.

Pentru a preveni problemele cauzate de vechea bibliotecă-logare a bunurilor comune, am scos-o. Acest lucru va provoca pluginului foarte vechi I2P-Bote (0.2.10 și de mai jos, semnat de HungryHobo) să se prăbușească în cazul în care sa activat IMAP. Remedierea recomandată este de a înlocui vechiul plug-in-I2P Bote cu cel actual semnat de str4d. Pentru mai multe detalii, a se vedea acest post.

Str4d a prezentat Lumea reală criptografică și a purtat un discurs cu privire la migrația noastră spre criptare, și diapozitivele sale sunt aici (pdf).

0.9.23 conține o varietate de erori remediate, și unele îmbunătățiri minore în I2PSnark

Salut I2P! Aceasta este prima lansare semnata de mine (str4d), după 49 lansari semnate de zzz. Acesta este un test important al redundanței noastre pentru toate lucrurile, inclusiv a oameniilor.

Curățenie

Cheia mea de semnare a fost în actualizările router-ului de peste doi ani (din 0.9.9), astfel dacă sunteți pe o versiune mai recentă a I2P această actualizare ar trebui să fie la fel de ușor ca oricare altă actualizare. Cu toate acestea, dacă rulați o versiune mai veche decât 0.9.9, dumneavoastră va trebui mai întâi să actualizați manual la o versiune mai recentă. Fișierele de actualizare pentru versiunea recentă pot fi descărcate de aici, și instrucțiuni despre cum să actualizați manual sunt furnizate aici. După ce actualizați manual, router-ul va găsi atunci și descărca actualizarea 0.9.23 ca de obicei.

În cazul în care instalați I2P printr-un manager de pachete, nu sunteți afectat de modificare, și puteți actualiza ca de obicei.

Detalii actualizare

Migrarea informaţilor routerului la noi, mai puternice Ed25519 semnături merge bine, cu cel puțin jumătate din reţeaua deja estimată a fi recodificată. Această versiune accelerează procesul de recodificare. Pentru a reduce abandonul de reţea, routerul dumneavoastră va avea o probabilitate mică de conversie la Ed25519 la fiecare repornire. Atunci când se recodifică, vă puteţi așteapta să vedeţi utilizarea lățimii de bandă mai mică pentru câteva zile, deoarece se reintegrează in rețea cu noua identitate.

Rețineți că acest lucru va fi ultima versiune care mai suportă Java 6. Vă rugăm să actualizați la Java 7 sau 8 cât mai curând posibil. Noi deja lucrăm pentru a face I2P compatibil cu viitorul Java 9, și o parte din această muncă se regăsește in această versiune.

Am făcut, de asemenea, unele îmbunătățiri minore în I2PSnark, și am adăugat o nouă pagină în consola router-ului pentru vizualizarea știrilor mai vechi.

Ca de obicei, vă recomandăm ca toți utilizatorii să actualizeze la această versiune. Cel mai bun mod de a ajuta rețeua și a rămâne securizat este de a rula ultima versiune.

0.9.22 cu erori remediate și startul migrației spre Ed25519

0.9.22 conține remedieri pentru i2psnark care se bloca inainte de finalizare, și începe migrarea informaţilor routerului la noi, mai puternice semnături Ed25519. Pentru a reduce abandonul de rețea, router-ul va avea doar o mică probabilitate de conversie la Ed25519 la fiecare repornire. Atunci când se recodifică, vă puteţi așteapta să vedeţi utilizarea lățimii de bandă mai mică pentru câteva zile, deoarece se reintegrează in rețea cu noua identitate. Dacă totul merge bine, vom accelera procesul de recodificare în următoarea lansare.
