Security issues found in nearly every system

A very serious issue has been found in modern CPUs which are found in nearly all sold computers worldwide, including mobiles. Those security issues are named "Kaiser", "Meltdown", and "Spectre".

The whitepaper for KAISER/KPTI can be found under KAISER.pdf, Meltdown under Meltdown.pdf, and SPECTRE under Spectre.pdf. Intels first response is listed under Intels response.

It is important for our I2P users to update your system, as long as updates are available. Windows 10 patches are availble from now on, Windows 7 and 8 will follow soon. MacOS patches are already available, to. Lots of Linux systems have a newer Kernel with a included fix available, too. Same applies to Android, which security fix of 2nd Jan 2018 includes a patch for this issues. Please update your systems as soon as available and possible.

0.9.30 升级到Jetty 9

0.9.30版本包含升级到Jetty 9和Tomcat 8的更新。 之前的版本不再被支持，也不会在即将到来的Debian Stretch和Ubuntu Zesty中提供。 路由器会将每个Jetty网站的jetty.xml设置文件迁移到新的Jetty 9设置中。 对于最近的未曾修改的设置的迁移应该会成功，但是对于修改过的或是非常老的设置则可能不会。 在升级之后请确认您的Jetty网站是否能够工作，如果需要帮助，请在IRC上联系我们。

一些插件与Jetty 9 不兼容，必须被升级。 下列插件已经升级，以适配0.9.30版本，您的路由器应在重启之后将其升级： i2pbote 0.4.6；zzzot 0.15.0。 下列插件（所列出的版本）无法在0.9.30版本工作。 请与相关的插件开发者联系，以便获知新版本的状态。 BwSchedule 0.0.36；i2pcontrol 0.11。

此次发布也支持将旧的（2014年及以前）DSA-SHA1隐藏服务迁移到更安全的EdDSA签名类型。 包括指引与FAQ在内的更多信息，请参见zzz.i2p。

注意：在非Android的ARM平台，例如树莓派，文件块数据库将在重启时升级，这也许会花上数分钟。 请保持耐心。

I2P-Bote 安全漏洞

I2P-Bote 0.4.5修复了一个在所有早期版本的I2P-Bote插件中 出现的安全漏洞。安卓平台的应用不受影响。

在为用户展示电子邮件时，大多数的字段都被escape编码过了。但是，附件的 文件名没有被编码，如果JavaScript被允许，可能会被用于执行浏览器 中的恶意代码。这些字段已被编码，另外，内容安全策略 也已在所有页面实行。

在I2P 0.9.29版本于二月中旬发布之后，所有的I2P-Bote用户都会在他们第一次 重启路由器时自动升级。但是，如果您计划在这期间使用I2P或者I2P-Bote， 为了安全，我们推荐您跟随安装页面上的指示 手动进行升级。

I2P-Bote 安全漏洞

I2P-Bote 0.4.4 修复了一个存在于 I2P-Bote 插件所有早期版本的漏洞。Android 版应用不受影响。

缺乏对于CSRF的保护意味着如果用户在运行I2P-Bote时在一个浏览器中 加载了一个恶意网站，并且允许了JavaScript，攻击者可以以用户的身份 在I2P-Bote中触发行为，例如发出信息。这也可能会允许 提取I2P-Bote中的私钥，但是对于这个可能尚且没有 验证。

在I2P 0.9.28于十二月中旬发布后，所有I2P-Bote的用户 都会在他们第一次重启路由器时自动更新。但是，如果您要在这期间使用I2P或者I2P Bote， 为了安全，我们建议您跟随安装页面上的指示 手动升级。如果您经常在 允许Javascript的时候浏览网页，并且打开了I2P-Bote， 您也需要考虑生成新的I2P-Bote地址。

0.9.26 包含加密学更新，Debian 包改进，以及缺陷修复。

0.9.26 对我们的加密程序库进行了一次重要升级，有新的地址簿订阅协议和签名，以及 Debian/Ubuntu 包的重要改进。

0.9.23 包含多个错误修复，以及一些对 I2PSnark 的小幅改善

管家

我的签名密钥已经在路由器更新中存在了两年多 (自从 0.9.9)，因此如果您在使用 I2P 的最新版本，此更新应该同以前的更新一样轻松。但如果您正在运行早于 0.9.9 的版本，您将需要先手动更新到一个最近的版本。最近颁布的更新文件可以在这里下载，并按照 这里提供的如何手动更新的指南进行。在您手动更新之后，您的路由器将照常查找和下载到 0.9.23 的更新。

如果您已通过一个软件包管理器来安装 I2P，您不会受到此更改的影响，并且可以照常更新。

更新细节

迁移 RouterInfos 到新的、更强的 Ed25519 签名很顺利，预计已有至少一半的网络转换密钥。此次发布加速了密钥更新的过程。为了减少网络客户端的流失，您的路由器将在每次重启时有较小的概率转换到 Ed25519。在它转换密钥后，可预期的两天内将有更低的带宽，因为它需要将新的身份重新集成到网络。

请注意，这是最后一个支持 Java 6 的发布版本。请尽快更新到 Java 7 或者 8。我们已经努力使 I2P 兼容即将到来的 Java 9，并且一些成果已经放在此次发布中。

我们也对 I2PSnark 做了一些细微的改进，并且在路由器控制台中加入了一个新的页面用以查看较旧的新闻。
