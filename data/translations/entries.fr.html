Flux d'actualités et mises à jour du routeur

I2P-Bote 0.4.4 fixes a security vulnerability present in all earlier versions of the I2P-Bote plugin. The Android app was not affected.

A lack of CSRF protection meant that if a user was running I2P-Bote and then loaded a malicious site in a browser with JavaScript enabled, the adversary could trigger actions in I2P-Bote on behalf of the user, such as sending messages. This might also have enabled extraction of private keys for I2P-Bote addresses, however no proof-of-exploit was tested for this.

All I2P-Bote users will be upgraded automatically the first time they restart their router after I2P 0.9.28 is released in mid-December. However, for safety we recommend that you follow the instructions on the installation page to upgrade manually if you plan on using I2P or I2P-Bote in the intervening time. You should also consider generating new I2P-Bote addresses if you regularly browse sites with JavaScript enabled while running I2P-Bote.

La version 0.9.26 contient des mises à jour de crypto, des améliorations de l'empaquetage pour Debian, et des corrections d'erreur

La version 0.9.26 contient une amélioration majeure concernant notre bibliothèque crypto native, un nouveau protocole d'abonnement au carnet d'adresses avec signatures, et des améliorations majeures concernant l'empaquetage Debian/Ubuntu.

Concernant la crypto, nous avons mis à niveau vers GMP 6.0.0 et nous avons ajouté le support de processeurs plus récents, ce qui accélérera considérablement les opérations de crypto. De plus, nous utilisons maintenant des fonctions GMP en temps constant afin d'empêcher des attaques côté canal. Par précaution, les changements concernant GMP sont activés seulement pour les nouvelles installations et les builds Debian/Ubuntu ; nous les inclurons avec les mises à jour intra-réseau à partir de la version 0.9.27.

Concernant les builds Debian/Ubuntu, nous avons ajouté des dépendances sur plusieurs paquets, incluant Jetty 8 et geoip, et enlevé le code équivalent qui était auparavant inclus.

Il y a aussi une collection de corrections d'erreurs, y compris une correction de bogue du minuteur qui causait de l'instabilité et des dégradations de performance au fil du temps. Comme d'habitude, nous recommandons que tous les utilisateurs mettent à niveau vers cette version. La meilleure façon d'aider le réseau à rester sécurisé consiste à exécuter la dernière version sortie.

Bonjour I2P ! Ceci est la première version sortie signée par moi (str4d), après 49 versions sorties signées par zzz. C'est un test important concernant notre redondance pour toutes les choses, y compris les personnes.

Tâche ménagère

Ma clé de signature est présente dans les mises à jour de routeur depuis plus de deux ans (depuis la v0.9.9), ainsi si vous êtes sur une version récente d'I2P cette mise à jour devrait être aussi facile que chaque autre mise à jour. Cependant, si vous exécutez une version précédente de la 0.9.9, vous devrez d'abord manuellement mettre à jour vers une version récente. Les fichiers de mise à jour pour des versions plus récentes peuvent être téléchargés ici, et les instructions concernant la façon de faire la mise à jour manuellement sont fournies ici. Une fois que vous aurez fait la mise à jour manuellement, votre routeur trouvera et téléchargera la v0.9.23 comme d'habitude.

Si vous avez installé I2P via un gestionnaire de paquets, vous n'êtes pas affectés par le changement et pouvez mettre à jour comme d'habitude.

Détails concernant cette mise à jour

La migration de RouterInfos vers de nouvelles signatures plus fortes, de type Ed25519, se passe bien avec une estimation d'au moins la moitié du réseau déjà re-verrouillée (rekeyed). Cette version accélère le processus de re-verrouillage (rekeying). Afin de réduire le taux de désabonnement de réseau (network churn), votre routeur aura une petite probabilité de conversion en Ed25519 à chacun de ses redémarrages. Lorsqu'il se re-verrouillera (rekey), attendez-vous à voir l'utilisation de bande passante diminuer pour deux ou trois jours tandis qu'il se réintègre dans le réseau avec sa nouvelle identité.

Notez que ceci sera la dernière version capable de supporter Java 6. Veuillez mettre à niveau vers Java 7 ou 8 dès que possible. Nous travaillons pour rendre I2P compatible avec le prochain Java 9, et une partie de ce travail est déjà présente dans cette version d'I2P.

Nous avons aussi effectué quelques améliorations mineures dans I2PSnark et avons ajouté une nouvelle page dans la console de routeur afin de pouvoir lire des actualités plus anciennes.

Comme d'habitude, nous recommandons que vous fassiez la mise à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'exécuter la dernière version sortie.

0.9.22 avec des corrections de bug et le début de la migration vers Ed25519

0.9.22 contient des corrections pour i2psnark qui bloquait avant l'accomplissement de téléchargements, et débute la migration des infos routeurs vers les nouvelles, plus fortes, signatures Ed25519. Afin de réduire le taux de désabonnement (anglais : churn), votre routeur aura seulement une petite probabilité de se convertir au Ed25519 lorsqu'il démarrera. Quand il se refournit en clé (anglais : rekeying), attendez-vous à voir l'usage en bande passante plus bas durant deux jours car il doit se réintégrer dans le réseau avec sa nouvelle identité. Si tout se passe bien, nous accélérerons le processus de refourniture de clés lors de la prochaine version qui sortira.
