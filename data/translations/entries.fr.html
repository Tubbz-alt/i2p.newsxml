Mise à niveau de 0.9.30 vers Jetty 9

0.9.30 contient une mise à niveau vers Jetty 9 et Tomcat 8. Les versions précédentes ne sont plus prises en charge et ne sont pas proposées dans les prochaines versions de Debian Stretch ni d'Ubuntu Zesty. Le routeur migrera le fichier de configuration jetty.xml de chaque site Web Jetty vers la nouvelle configuration Jetty 9. Cela devrait fonctionner pour les configurations récentes, non modifiées, mais peut-être pas pour les configurations modifiées ou très anciennes. Vérifiez que votre site Web Jetty fonctionne après la mise à niveau et contactez-nous sur IRC si vous avez besoin d'assistance.

Plusieurs greffons ne sont pas compatibles avec Jetty 9 et doivent être mis à jour. Les greffons suivants ont été mis à jour pour fonctionner avec 0.9.30 et votre routeur devrait les mettre à jour après redémarrage : I2pbote 0.4.6; zzzot 0.15.0. Les greffons suivants (versions actuelles listées) ne fonctionneront pas avec 0.9.30. Contactez le développeur approprié du greffon pour connaître l'état des nouvelles versions : BwSchedule 0.0.36; i2pcontrol 0.11.

Cette version prend aussi en charge la migration des anciens (2014 et antérieurs) services cachés DSA-SHA1 vers le type de signature plus sécurisé EdDSA. Voir zzz.i2p pour de plus amples informations, incluant guide et FAQ.

Notez : sur les plateformes ARM non Android telles que le Raspberry Pi, la base de données blockfile se mettra à jour lors du redémarrage, ce qui pourrait prendre quelques minutes. Soyez patient.

Comme d'habitude, nous recommandons que vous mettiez à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'exécuter la dernière version.

I2P-Bote 0.4.5 corrige une faille de sécurité présente dans toutes les versions précédentes du greffon I2P-Bote. L'appli pour Android n'était été affectée.

Lors de l'affichage de courriels à l'utilisateur, la plupart des champs étaient échappés. Cependant, les noms de fichier des pièces jointes n'étaient pas échappés et pouvaient être utilisés pour exécuter du code malveillant dans un navigateur où JavaScript était activé. I2P-Bote 0.4.4 corrige une faille de sécurité présente dans toutes les versions précédentes du greffon I2P-Bote. L'appli pour Android n'était été affectée.

Un manque de protection CSRF signifiait que si un utilisateur exécutait I2P-Bote et chargeait ensuite un site malveillant dans un navigateur avec JavaScript activé, l'adversaire pouvait déclencher des actions dans I2P-Bote au nom de l'utilisateur, comme envoyer des messages. Cela pourrait aussi avoir permis l'extraction des clés privées d'adresses d'I2P-Bote, cependant aucune preuve d'exploitation n'a été testée pour cela.

Tous les utilisateurs d'I2P-Bote seront automatiquement mis à niveau la première fois qu'ils redémarreront leur routeur après parution d'I2P 0.9.28 à la mi-décembre. Cependant, pour des raisons de sécurité, nous recommandons que vous suiviez les instructions sur la page d'installation afin de la mettre à niveau manuellement si vous prévoyez utiliser I2P ou I2P-Bote entre temps. Vous devriez aussi envisager de générer de nouvelles adresses I2P-Bote si vous naviguez régulièrement sur des sites avec JavaScript activé tout en exécutant I2P-Bote.

0.9.26 contient des mises à jour cryptographiques, des améliorations de l'empaquetage pour Debian et des correctifs de bogues

0.9.26 contient une mise à niveau importante de notre bibliothèque cryptographique native, un nouveau protocole d'abonnement de carnet d'adresses avec signatures et des améliorations importantes de l'empaquetage Debian et Ubuntu.

Concernant la cryptographie, nous sommes passé à GMP 6.0.0 et nous avons ajouté la prise en charge de processeurs plus récents ce qui accélérera considérablement les opérations cryptographiques. De plus, nous utilisons maintenant des fonctions GMP en temps constant afin d'empêcher les attaques par canal auxiliaire. Par précaution, les changements dans GMP seront seulement activés pour les nouvelles installations et les versions Debian ou Ubuntu ; nous les inclurons avec les mises à jour intraréseau dans la version 0.9.27.

Pour les versions Debian ou Ubuntu, nous avons ajouté des dépendances vers plusieurs paquets, incluant Jetty 8 et geoip et nous avons supprimé le code intégré équivalent.

Il y a aussi une collection de correctifs de bogues, y compris un correctif de bogue de minuteur qui causait de l'instabilité et une dégradation des performances au fil du temps. Comme d'habitude, nous recommandons que tous les utilisateurs mettent à jour I2P vers cette version. 0.9.23 contient divers correctifs de bogues et des améliorations mineures dans I2PSnark

Bonjour I2P ! Ceci est la première version signée par moi (str4d), après 49 versions signées par zzz. C'est un test important de notre redondance pour tout, y compris les personnes.

Gestion interne

Ma clé de signature est présente dans les mises à jour du routeur depuis plus de deux ans (depuis 0.9.9), donc si vous utilisez une version récente d'I2P, cette mise à jour devrait être aussi facile que n'importe quelle autre mise à jour. Cependant, si vous exécutez une version antérieure à 0.9.9, vous devrez d'abord passer manuellement à une version récente. Les fichiers de mise à jour des versions récentes peuvent être téléchargés ici et les instructions de mise à jour manuelle se trouvent ici. Une fois que vous aurez mis votre version à jour manuellement, votre routeur trouvera et téléchargera alors la mise à jour 0.9.23 comme d'habitude.

Si vous avez installé I2P grâce à un gestionnaire de paquets, vous n'êtes pas affectés par le changement et pouvez mettre votre version à jour comme d'habitude.

Détails sur la mise à jour

La migration d'InfosRouteur vers de nouvelles signatures Ed25519 plus robustes se passe bien et il est estimé qu'au moins la moitié du réseau utilise déjà les nouvelles signatures. Cette version accélère ce processus. Afin de réduire le taux de désabonnement du réseau, votre routeur aura une faible probabilité de conversion à Ed25519 lors de chaque redémarrage. Lorsqu'il utilisera les nouvelles signatures, attendez-vous à voir une utilisation moindre de la bande passante pendant quelques jours alors qu'il réintègre le réseau avec sa nouvelle identité.

Noter que cette version sera la dernière à prendre Java 6 en charge. Veuillez mettre votre Java à jour vers Java 7 ou 8 dès que possible. Nous nous efforçons déjà de rendre I2P compatible avec le futur Java 9 et une partie de ce travail est présente dans cette version.

Nous avons aussi effectué quelques améliorations mineures dans I2PSnark et avons ajouté une nouvelle page dans la console du routeur afin de visualiser les anciennes nouvelles.

Comme d'habitude, nous recommandons que vous mettiez à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'exécuter la dernière version.
